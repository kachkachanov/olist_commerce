# olist_commerce
E-commerce data analysis: SQL, Python, product metrics, A/B testing
Analysis of Brazilian e-commerce platform Olist. 
The project covers revenue dynamics, product categories, 
delivery performance, customer retention and A/B testing 
using SQL, Python (pandas, scipy) and DataLens dashboard.
Stack: PostgreSQL • Python • pandas • scipy • Yandex DataLens

# Все таблицы 
1. customers — покупатели. Кто делает заказы, откуда они (город, штат), уникальные пользователи. Основа для клиентского анализа.
2. orders — заказы. Главная таблица проекта. Содержит весь жизненный цикл заказа: когда купили, когда одобрили, когда отправили, когда доставили. Основа для анализа динамики и доставки.
3. order_items — позиции заказов. Что именно купили в каждом заказе, у какого продавца, по какой цене. Один заказ может содержать несколько позиций.
4. payments — платежи. Чем оплатили (карта, ваучер, боливо), в сколько платежей, на какую сумму. Один заказ может быть оплачен несколькими способами.
5. products — товары. Категории, физические характеристики (вес, размеры). Нужна для анализа по категориям.
6. sellers — продавцы. Откуда продавец, его местоположение. Olist — это маркетплейс, поэтому продавцов много.
7. reviews — отзывы покупателей. Оценка от 1 до 5, текстовый комментарий. Показывает удовлетворённость клиентов.
8. geolocation — координаты по zip-коду. Нужна для географической визуализации на карте — джойнишь с customers или sellers по zip_code.
